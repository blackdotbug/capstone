ABOUT THE PROJECT

What Got Us Here

Making Progress in the Fight

There's been a lot going on in the fight against internet toxicity. Feminist advocacy groups like UltraViolet are putting a lot of work into reports and analysis of the current state of things. Their findings are not great.

Oh the Humanity

Keeping humans in the loop in moderating online content is both necessary and challenging. Humans come with bias, but so do models built by humans. And the impact to individuals who do the work of moderating toxic content online can have serious detrimental effects.

Machine Learning to the Rescue

Natural Language Processing models are frequently used to classify text. What dataset you use to train them can have a lot of impact on their effectiveness. A recent project out of GESIS in Germany found that by carefully constructing a training dataset, automated measures for detecting sexism can be improved.

Disruptors Gonna Disrupt

While lots of progress is being made, there's still a lot to do. That hasn't stopped tech entrepreneurs from trying to capitalize on this emerging market of automated content moderation. Before these startups do more harm than good by introducing poorly trained models into the wild, let's keep working the problem.

THE ANALYSIS

What We Did

The Call Me Sexist, But Dataset

where we started

The dataset we used to train our model was a sample of 10,000 tweets from the dataset published with the "Call me sexist, but...": Revisiting Sexism Detection Using Psychological Scales and Adversarial Samples paper.

The authors combined known datasets (Hostile Sexism Tweets and Other (non-sexist) Tweets from Hateful Symbols or Hateful People? Predictive Features for Hate Speech Detection on Twitter and Benevolent Sexism Tweets from When does a compliment become sexist? Analysis and classification of ambivalent sexism using twitter data) with their own data (Call Me Sexist Tweets) and statements they pulled from psychological scales designed to measure sexism in individuals (Sexism Scales)

They added another subset of datapoints they reference as "Adversarial". In the paper they describe the process used for generating this data: "We challenged crowdworkers to remove as much of the sexism as possible from the sexist examples, so as to produce adversarial examples which have minimal lexical changes, but that are non-sexist."

All of the datapoints were labeled as sexist or not sexist, and samples from each dataset were annotated and labels potentially modified by crowdsourced workers using a sexism codebook the authors defined.

Our Model

what we made

We used tensorflow and keras to build and compile a Convolutional Neural Network model, taking advantage of keras' TextVectorization layer in the tf-idf output mode to convert text to numeric vectors. Our output layer used a sigmoid activation function to produce probablistic predictions.

We used several different types of layers and trained our model in three epochs. Once trained our model evaluated on the test split of the dataset with an accuracy of 0.8716

We used nearly identical text preprocessing functions as found in the github repository associated with source dataset.

Finally, we made an end-to-end model that can accept our preprocessed text strings by adding an additional input layer and the TextVectorization layer we adapted to the source dataset. This model we saved and deployed to this website to be used to score reddit comments.

The Reddit Data

how it's going

Using the Reddit API we set up a streaming endpoint to capture real-time comments from any given subreddit, from which we pulled the comment body and ran it through our preprocessor (additionally removing any blockquotes). We stored the processed text and our model's prediction in a Mongo Atlas database.
